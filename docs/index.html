Features

Everything you need to query Qobuz

From instant search to playlist management, mcp-qobuz gives your AI client first-class access to the Qobuz catalogue and your personal library.

🔍

Eleven tools out of the box

Search tracks, albums, artists, and playlists; fetch detail pages; list your favourites; and manage playlists — all from any MCP client.

🔐

Two auth paths

macOS Keychain via createKeychainStore() — shared with qobuz-cli and the Raycast extension — or QOBUZ_TOKEN + QOBUZ_APP_ID env vars for headless use.

🛡️

Safety-gated writes

create-playlist, add-to-playlist, and update-playlist-description require confirm: true to prevent accidental mutations to your library.

🎧

Now-playing from the desktop app

Reads the Qobuz desktop app's local player state file on macOS. Ask your AI what's playing and get a full track object with a deep link back to the player.

🔗

Deep links on every result

Every tool response includes an open.qobuz.com URL so results are one click away from the player — no copy-pasting IDs.

📦

Powered by @kud/qobuz

Built on the same @kud/qobuz core library used across the kud music toolchain — qobuz-cli, the Raycast extension, and this server all share one authenticated client.

Quick Start

Up and running in minutes

Add mcp-qobuz to your MCP client config. No build step — runs directly via npx.

1 Keychain auth (macOS — shared credentials) 
// .mcp.json
{
  "mcpServers": {
    "mcp-qobuz": {
      "command": "npx",
      "args": ["-y", "@kud/mcp-qobuz"]
    }
  }
}

Credentials are read automatically from the macOS Keychain. Log in once with qobuz-cli or the Raycast extension and this server picks them up.

2 Env var auth (headless / remote) 
// .mcp.json
{
  "mcpServers": {
    "mcp-qobuz": {
      "command": "npx",
      "args": ["-y", "@kud/mcp-qobuz"],
      "env": {
        "QOBUZ_TOKEN": "your-token",
        "QOBUZ_APP_ID": "your-app-id"
      }
    }
  }
}

Env vars take precedence over the Keychain when both are present. Useful for CI, remote servers, or non-macOS hosts.

Tool Reference

Eleven tools, two modes

Read tools work immediately after auth. Write tools require confirm: true — a deliberate speed bump before mutating your library.

search Search Qobuz for albums, artists, and/or tracks. Returns results with Qobuz deep links. Filter by type: albums, artists, tracks, or all. Read
get-track Fetch full track details by numeric ID, including deep link back to the Qobuz player. Read
get-album Fetch full album details by string ID (e.g. '0060253786997'), including tracklist and deep link. Read
get-artist Fetch artist biography and discography by numeric ID, including deep link. Read
get-playlist Fetch a playlist and its full track listing by numeric ID. Read
list-playlists List the authenticated user's Qobuz playlists. Returns up to 50 by default (configurable up to 500). Read
list-favourites List your favourited albums, artists, or tracks. Pass type to select which collection to browse. Read
now-playing Show the track currently playing in the Qobuz desktop app. Reads the local player state file written by the desktop app. macOS only
create-playlist Create a new Qobuz playlist with optional description and visibility. Requires confirm: true. Write
add-to-playlist Add one or more tracks to an existing playlist by ID. Requires confirm: true. Write
update-playlist-description Update a playlist's description text. Requires confirm: true. Write
